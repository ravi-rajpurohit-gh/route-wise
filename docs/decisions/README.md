# Architecture Decision Records

Architecture Decision Records preserve the context and consequences of
important technical choices.

## Status values

- **Proposed:** Under active evaluation.
- **Accepted:** Current direction.
- **Superseded:** Replaced by a later ADR.
- **Rejected:** Evaluated but not selected.

Copy the structure of an existing ADR. Record context, decision, consequences,
and alternatives. ADRs explain why a choice was made rather than repeating
implementation details visible in code.
